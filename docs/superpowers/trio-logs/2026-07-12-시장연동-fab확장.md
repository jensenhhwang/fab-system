# 트리오 기획 세션 — 시장 연동 + FAB 확장
**날짜:** 2026-07-12  
**트리거:** "기획팀에서 방안을 세워서 가져와줘"  
**의제:** 창고 capacity 점유가 낮은 원인 파악 + HBM/DRAM/NAND 시세 연동 + FAB 확장 방안

---

## 문제 정의

현재 WH-A 3%, WH-B 4% 점유는 **시드 데이터 스케일 문제**.  
WH-A 7,000 pallet 용량 대비 실제 씨드된 재고는 200-300 pallet 수준 (실제 팹 대비 1/15~1/20 스케일).  
해결 방향: ProcessUsage.monthlyQty를 시장 수요 기반으로 스케일업 + 외부 시세 데이터 연동.

---

## 🎨 크리 제안

1. **살아있는 팹 — 반도체 시장 지수 ↔ 소비량 자동 연동 엔진**  
   HBM/DRAM/NAND 시세를 당겨와 ProcessUsage 소비량을 자동 스케일링. 시세 사이클에 맞춰 창고 점유율이 실시간 변동.

2. **HBM +X% 대응 시뮬레이터 — `/simulation/market` 신설**  
   제품별 수요 슬라이더 → 공정 가동률, 자재 소비량, 창고 포화 시점 자동 계산. "우리 팹이 이 수요를 언제까지 버티냐"를 데이터로 제시.

3. **AI FAB 확장 어드바이저**  
   임계 신호(창고 포화 90일 내) 감지 시 "WH-E 5000 pallet 증설 시뮬레이션" 자동 권고. 클릭 한 번으로 가상 창고 추가 후 capacity 재검증.

---

## ⚙️ 패브 검토

### 아이디어 1 — 시장 지수 연동 엔진
**판정: ✅ 가능**  
- TrendForce 등 공개 API 없으나 `/api/market-sync` cron + 스크래핑으로 구현 가능  
- `materials` 컬렉션에 `marketPrice: number`, `demandFactor: number` 필드 추가  
- `ProcessUsage.monthlyQty`에 demandFactor를 곱해 소비량 동적 조정  
- MongoDB cron 또는 Vercel Cron Functions (일 1회)

### 아이디어 2 — 수요 시나리오 시뮬레이터
**판정: ⚠️ 수정 필요**  
- 기존 `/simulation`은 단일 자재 입고 A/B안 비교 — 충돌 방지를 위해 `/simulation/market` 서브루트 신설  
- 입력: 제품별(HBM/DRAM/NAND) 수요 변화율 슬라이더  
- 계산: `processUsage` × 제품 비율 × demandFactor → 공정별 소비량 증분  
- 출력: 창고별 예상 점유율, 포화 시점 (현재 재고 ÷ 증가된 일소비량)  
- 기술적으로 완전 구현 가능, 복잡도 중간

### 아이디어 3 — FAB 확장 어드바이저
**판정: ✅ 가능**  
- `warehouse` 컬렉션에 `status: "virtual" | "active"` 필드 추가  
- 가상 창고는 실운영에 영향 없이 capacity 예측에만 사용  
- Risk 페이지 경고 로직 재활용 — 기존 인프라 그대로

**우선순위:**
1. 시드 데이터 스케일업 (ProcessUsage.monthlyQty × 15-20배율 마이그레이션 스크립트)  
2. `/simulation/market` 시나리오 시뮬레이터  
3. 시장 지수 연동 cron  
4. FAB 확장 어드바이저

### 사용자 흐름
1. 대시보드 시장 지수 카드 클릭 → `/simulation/market`
2. 슬라이더 조정 → 실시간 공정·창고 영향 표시
3. "WH-C ⚠️" 클릭 → Risk 페이지 해당 항목 딥링크
4. "가상 WH-E 시뮬레이션" → 증설 후 여유 확인
