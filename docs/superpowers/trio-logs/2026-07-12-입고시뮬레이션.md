# 트리오 세션 — 입고 시뮬레이션

**날짜:** 2026-07-12  
**기능 요청:** 자재 선택 → 입고 수량/날짜 입력 → DOH 변화 예측 그래프

---

## 크리

저는 크리인데요, 입고 시뮬레이션이라는 주제를 받으니까 단순히 "수량 입력하고 DOH 보여주기"보다 훨씬 더 흥미로운 방향이 보여요.

**1. 공급망 도미노 시뮬레이터**
입고 수량을 바꾸면 그게 연관된 공정(P01~P10)에 어떤 연쇄 효과를 내는지 도미노처럼 시각화하면 어떨까요? EUV PR 입고를 늘리면 HBM 라인 DOH가 얼마나 버티는지 한눈에 보이는 거예요.

**2. AI 발주 타이밍 추천**
과거 소비 패턴이랑 공정 스케줄을 분석해서 "이 자재는 다음 주 화요일에 주문하면 딱 맞아요"라고 달력에 핀 꽂아주는 시스템이요.

**3. 멀티 시나리오 비교 시뮬레이션**
"A안: 500kg 입고 vs B안: 1000kg 입고" 두 시나리오를 나란히 놓고 DOH 곡선을 비교하면 어떨까요? 발주 결정 회의 때 바로 의사결정할 수 있어요.

---

## 패브

저는 패브인데요, 크리 아이디어들 실제로 우리 시스템 기준으로 따져볼게요.

**⚠️ 1. 공급망 도미노 시뮬레이터 — 수정 필요**
`processUsage` 컬렉션에 공정별 소비량 데이터가 있어서 계산은 가능하지만, 3D 도미노 애니메이션은 오버스펙.
수정안: 입고 자재 → 해당 자재를 쓰는 공정 목록 자동 표시 → 각 공정의 DOH 변화를 바 차트로.

**⚠️ 2. AI 발주 타이밍 추천 — 수정 필요**
AI 학습 데이터 부족, 구매 승인 프로세스와 충돌.
수정안: DOH 수식 기반 "재고 소진 예상일" 캘린더 표시. `inventory` + `processUsage`로 즉시 구현 가능.

**✅ 3. 멀티 시나리오 비교 — 가능**
프론트엔드에서 두 수량값으로 DOH 계산 후 나란히 표시. 신규 API 최소화.
요구사항: `/api/simulation/doh` POST, 입력값 `{ materialId, qty_a, qty_b, date }`.

## 엑스

저는 엑스인데요, 패브가 정리한 범위로 화면 설계 들어갈게요.

**레이아웃 — `/simulation` 페이지**

```
┌─────────────────────────────────────────────────────┐
│ 상단: 자재 선택 드롭다운 + 날짜 범위 피커            │
├─────────────────┬───────────────────────────────────┤
│ 좌측 (40%)      │ 우측 (60%)                         │
│                 │                                   │
│ A안 수량 입력   │ DOH 비교 라인 차트                 │
│ [____] kg       │ (A안 파란선 / B안 주황선)           │
│                 │                                   │
│ B안 수량 입력   │ 소진 예상일 배지                    │
│ [____] kg       │ A안: 2026-08-12 / B안: 2026-09-04  │
│                 │                                   │
│ [시뮬레이션 실행] │ 영향 공정 목록 (P03, P07 등)       │
└─────────────────┴───────────────────────────────────┘
```

**주요 컴포넌트:** 자재 드롭다운, DOH 라인 차트, 소진 예상일 배지(빨강/주황/초록), 영향 공정 목록

**사용자 흐름:** 자재 선택 → A/B 수량 입력 → 실행 클릭 → 차트 즉시 업데이트 → 공정 하이라이트

**기존과 다른 점:** 최초의 "미래 예측" 인터랙티브 화면. 실행 전 차트 비어있다가 클릭하면 채워지는 패턴.
