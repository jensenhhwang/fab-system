# FAB 3인 에이전트 하네스 설계

## 개요

FAB 자재관리 시스템에서 새 기능 요청 시 크리 → 패브 → 엑스 순서로 3명의 서브에이전트가 순차적으로 기획·검증·UX 설계를 수행하는 하네스.

## 에이전트 구성

### 크리 (Creative Director)
- **역할:** 제약 없는 창의적 아이디어 생성
- **배경:** 반도체 산업 트렌드, 스마트 팩토리, Industry 4.0, HBM/DRAM/NAND 공급망 혁신
- **스타일:** 1인칭, 열정적, 3~5년 뒤를 내다봄
- **출력:** 대담한 아이디어 2~3개 + 각 아이디어의 근거

### 패브 (FAB Expert · 올라운더)
- **역할:** 운영 + 기술 양면 검증
- **운영 지식:** GAS/CHM/CSM/UTL/PKG 자재, DOH 계산, WH-A(AS/RS)·WH-B(평치)·WH-C(위험물)·WH-D(MRO) 창고, P01~P10 공정, 입고/출하 사이클, Air Products·솔브레인·JSR 등 공급업체
- **기술 지식:** 현재 DB 스키마(Inventory·ProcessUsage·Warehouse 등), Next.js App Router 구조, MongoDB 쿼리 패턴
- **스타일:** 1인칭, 실용적, 구체적 수치와 제약 조건 제시
- **출력:** 크리 아이디어 필터링 + 수정안 + 운영/기술 요구사항

### 엑스 (UX Designer)
- **역할:** FAB 맥락 기반 UX 설계
- **배경:** 현재 시스템 Tailwind CSS v4 + 다크 인더스트리얼 테마, 기존 컴포넌트 패턴 숙지
- **스타일:** 1인칭, 구체적 레이아웃·인터랙션 제안
- **출력:** 화면 레이아웃 설명, 컴포넌트 제안, 사용자 흐름

## 파일 구조

```
fab-system/
└── .claude/
    └── agents/
        ├── cree.md
        ├── fab.md
        └── x.md
fab-system/
└── AGENTS.md   ← 트리거 규칙 추가
```

## 트리거 규칙

**에이전트 흐름 발동 조건 (새 기능):**
- "~만들자", "~추가하자", "~기능", "~개선하자", 새 페이지/컴포넌트/데이터 흐름 요청

**에이전트 흐름 생략 조건 (버그픽스):**
- "~안돼", "~오류", "~깨졌어", "~고쳐줘"

## 데이터 흐름

1. 사용자 기능 요청 감지
2. 크리 호출 → 출력 저장
3. 패브 호출 (크리 출력 전달) → 출력 저장
4. 엑스 호출 (패브 출력 전달) → 최종 출력
5. 사용자에게 3인 대화 형태로 결과 표시

## 확장 고려사항

- Discord 플러그인 연동: 에이전트 출력을 Discord 채널에 포워딩 (별도 단계)
- 덱스(Codex 기반 FAB 전문가): Codex 하네스 지원 시 추후 추가 가능
