# FAB System — 프로젝트 비전

## 핵심 목표

**ERP → WMS → MES를 실제로 연결하는 통합 제조 운영 플랫폼을 구축한다.**

단순한 개별 시스템이 아니라, 세 레이어가 데이터를 주고받으며 하나의 흐름으로 작동하는 구조를 만드는 것이 1차 목표다.

```
ERP (구매·발주·재무)
  ↕
WMS (창고·입출고·로케이션)
  ↕
MES (생산지시·공정·실적)
```

---

## 1차 목표: ERP-WMS-MES 연동

| 레이어 | 역할 | 현황 |
|--------|------|------|
| WMS | 창고 입출고, 로케이션, 재고 추적 | ✅ 구축 완료 |
| ERP | 발주, 입고, 자재 마스터, 재고 평가 | 🔄 시뮬레이터 진행 중 |
| MES | 생산지시, 공정 실적, BOM 소비 | ⬜ 미착수 |

**WMS-MES 연동의 전제조건은 ERP 연동이다.**  
자재가 어디서 왔는지(ERP 발주), 어디에 있는지(WMS 위치), 어디서 소비되는지(MES 공정)가 하나의 데이터 체계로 연결되어야 한다.

---

## 2차 목표: 디지털 트윈

가상 환경에서 생산이 **실시간으로 계속 돌아가는** 디지털 트윈을 구현한다.

- 시간 가속(타임 액셀레이터): 1일 = 수초 단위로 가상 운영 진행
- 실 DB에 `simulated: true` 마킹으로 실데이터와 시뮬 데이터 공존
- 가상 사이클: 자재 소비 → ROP 도달 → 자동 발주 → 입고 → 재소비
- 공급 지연, 부분 입고, 재발주 등 실제 공급망 리스크를 가상에서 미리 관찰

궁극적으로는 실제 공장 데이터와 가상 시뮬레이션이 나란히 흐르며, 의사결정을 시뮬로 먼저 검증하는 구조를 목표로 한다.

---

## 구현 로드맵

### Phase 1 — ERP 시뮬레이터 (진행 중)
- 발주-입고 가상 사이클 (FEFO 소비, ROP 자동 발주)
- 타임 액셀레이터 (SSE 실시간 피드)
- 체크포인트 기반 시간 되감기

### Phase 2 — ERP 실연동
- 실제 ERP 시스템(혹은 ERP 역할 모듈) 연결
- 발주 승인, 입고 확인 워크플로우

### Phase 3 — MES 구축
- 생산지시 발행, BOM 기반 자재 소비
- WMS 피킹과 MES 투입 연결
- 공정 실적 기록 → ERP 원가 반영

### Phase 4 — 완전한 디지털 트윈
- 실공장 + 가상공장 병렬 운영
- 시나리오 기반 의사결정 지원 (What-if 분석)
