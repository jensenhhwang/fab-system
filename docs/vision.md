# FAB System — 프로젝트 비전

## 핵심 목표

**ERP → WMS → MES를 실제 데이터와 승인 흐름으로 연결하는 통합 제조 운영 플랫폼을 구축한다.**

```text
ERP (계획·발주·재무)
  ↕
WMS (창고·입출고·위치·로트)
  ↕
MES (작업지시·피킹·공정 실적)
```

## 현재 상태

| 레이어 | 역할 | 현황 |
|---|---|---|
| WMS | 입출고, 위치, 로트·용기 재고 추적 | ✅ Phase 1 구현 |
| ERP/SCM | 자재 마스터, 승인 공급사, 리드타임 범위, 발주 계획 | 🔄 조달 기준 Phase 1, 승인·실입고 연동 필요 |
| MES | 작업지시, BOM, FEFO 피킹, 공정 준비도 | ✅ Phase 1 구현 |

자재 ID를 운영 정보의 기준축으로 사용한다. 자재 운영 허브에서 전체·창고별 재고, Lot 품질, 공정 사용량, BOM, 승인 공급사와 리드타임을 하나의 흐름으로 조회하고, 조달 기준 수정은 SCM과 What-if에 동일하게 반영된다. 공정별 사용량을 일사용량과 DOH의 기준으로 사용하고, WMS 로트 재고가 MES 피킹과 movement 기록으로 연결된다. 다음 핵심 과제는 ERP 승인·실입고와 MES 공정 실적·원가를 같은 흐름에 연결하는 것이다.

## 디지털 트윈 방향

과거의 랜덤 자동운전과 타임 액셀러레이터는 현행 목표에서 제외한다. 디지털 트윈은 실제 운영 스냅샷에 사용자가 명시한 조건을 적용하는 **설명 가능하고 재현 가능한 What-if** 도구로 발전시킨다.

- 수요 변화, 입고 계획, 공급 지연 조건을 명시적으로 입력
- 동일한 입력은 동일한 결과를 반환
- 결과를 `SIMULATION`으로 표시해 운영 데이터와 구분
- 실제 반영 전에 재고, 결품, 창고 Capacity 영향을 비교
- 입력, 결과, 검토 및 승인 이력을 보존
- 통상 리드타임과 보수적 최대 리드타임을 분리해 발주 가능 범위를 설명

## 로드맵

### Phase 1 — WMS·MES 연결 기반 ✅

- 위치·로트·용기 기반 WMS와 FEFO 출고
- BOM 기반 MES 작업지시와 자재 피킹
- 공정 사용량에서 재고 DOH까지 단일 계산 흐름

### Phase 2 — ERP 승인·실연동

- 승인 공급사와 주/보조 조달 역할 관리
- 재고 목록에서 진입하는 자재 단위 종합 운영 허브
- 최소·기준·최대·현재 예상 리드타임 관리
- 발주 요청·승인·발행 워크플로우
- 실제 입고 확인과 WMS 반영
- 공급사, 리드타임, 단가 및 재고 평가 연결

### Phase 3 — MES 실적 고도화

- 공정 실적·수율·불량 기록
- WMS 피킹과 실제 투입·반납 연결
- ERP 원가와 생산 실적 반영

### Phase 4 — 설명 가능한 디지털 트윈

- 실공장 스냅샷과 What-if 결과 병렬 비교
- 수요·입고·지연 조건 기반 의사결정 지원
- 시나리오의 입력·결과·승인 이력 보존
